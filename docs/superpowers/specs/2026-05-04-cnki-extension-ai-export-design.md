# 知网浏览器插件 AI 分析包设计

## 背景

用户已经可以在知网中完成检索，真正的痛点不是“如何检索”，而是检索结果数量过大。面对数百到上千篇论文，人工逐篇复制题录、进入详情页查看摘要、整理关键词，再归纳研究类型、反复观点、代表论文、已有贡献、共同不足和研究空白，会消耗大量时间。

第一版产品优先解决“把知网检索结果整理成 AI 可读材料”的问题。插件在用户已经登录知网的浏览器页面中运行，采集论文列表和详情页可见信息，导出结构化文本、CSV、JSONL 和分析提示词。用户再把这些文件上传到 ChatGPT、DeepSeek 等网页 AI 工具中做文献综述前期分析。

## 第一版目标

构建一个 Chrome / Edge Manifest V3 浏览器插件，完成：

1. 从知网检索结果页采集当前页可见论文列表。
2. 解析知网论文详情页中的摘要、关键词、基金资助、专辑、专题、分类号等字段。
3. 在浏览器本地临时保存采集进度。
4. 对重复论文去重，保留已经补全的详情字段。
5. 导出 AI 分析包，方便用户上传到 ChatGPT 或 DeepSeek。

第一版不做本地工作台，不接 AI API，不批量下载全文，不绕过知网登录、验证码或付费限制。

## 数据字段

每条论文记录包含：

- 论文 ID
- 题名
- 作者
- 来源
- 发表时间
- 数据库类型
- 被引次数
- 下载次数
- 详情页链接
- 摘要
- 关键词
- 基金资助
- 专辑
- 专题
- 分类号
- 采集时间
- 采集状态

## 导出文件

插件导出 zip 包，包含：

- `papers.csv`：适合人工检查和表格软件处理。
- `papers.jsonl`：适合程序继续处理。
- `papers_for_ai.md`：适合直接上传给 AI 阅读的总文件。
- `batch_001.md` 等：按批次拆分，降低网页 AI 单次上下文压力。
- `analysis_prompt.md`：要求 AI 回答用户关心的九个问题。
- `merge_prompt.md`：用于合并多个批次的分析结果。
- `README.txt`：导出包使用说明。

## 用户流程

1. 用户在浏览器中打开知网并登录。
2. 用户使用知网自己的检索功能得到论文列表。
3. 用户打开插件弹窗，点击“采集当前页”。
4. 插件读取当前检索结果页的可见论文，并保存到浏览器本地。
5. 用户翻页后继续点击“采集当前页”，直到采集到需要的范围。
6. 用户点击“导出 AI 分析包”。
7. 用户把提示词和批次 Markdown 上传到 ChatGPT 或 DeepSeek，完成归纳总结。

## 重要边界

- 不保存、读取或代填知网账号密码。
- 不绕过验证码、登录限制、付费限制或下载限制。
- 不批量下载 PDF / CAJ 全文。
- 不直接调用 ChatGPT、DeepSeek 或其他模型 API。
- 不承诺无人值守采集上千篇论文；遇到登录失效、验证码或页面异常时，应由用户手动处理。

## 后续增强方向

- 自动按详情页链接逐篇补全摘要和关键词。
- 多页采集、暂停、继续和失败重试。
- 采集进度更细的可视化。
- 导出前的筛选、排序和去重确认。
- 将浏览器插件与本地工作台连接，做更长上下文的本地整理。
