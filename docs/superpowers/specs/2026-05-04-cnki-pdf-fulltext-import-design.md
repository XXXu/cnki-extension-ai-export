# 知网 PDF 全文导入与分批分析设计

## 背景

当前插件已经可以从知网检索结果页采集题录、摘要和关键词，并导出给 ChatGPT 或 DeepSeek 做初步文献综述分析。摘要适合处理上千篇论文的宏观归纳，但如果要判断一篇论文的论证质量、研究方法、材料基础、创新点和不足，只看摘要不够，需要引入全文。

本设计增加 PDF 全文导入能力，但不让插件自动批量下载知网全文。用户仍然在知网内按账号权限下载 PDF，插件只处理用户主动选择的本地 PDF 文件。

## 目标

1. 支持用户一次选择多个 PDF 全文文件。
2. 从 PDF 中提取可读文本。
3. 尽量自动匹配到已采集的论文记录。
4. 导出分析包时，生成快速摘要分析包和全文增强分析包。
5. 保持“快速看全局，全文增强做深度判断”的工作流，避免让用户把摘要和全文重复分析两遍。

## 非目标

1. 不自动批量下载知网 PDF。
2. 不处理 CAJ 文件。
3. 不绕过知网登录、验证码、付费或下载限制。
4. 不直接调用 ChatGPT、DeepSeek 或其他 AI API。
5. 第一版不做复杂的手动匹配编辑界面，只做自动匹配和未匹配提示。

## 推荐工作流

1. 用户在知网检索观点。
2. 插件点击“采集当前页并补全摘要”。
3. 用户根据摘要分析或个人判断下载一批重点论文 PDF。
4. 插件点击“导入全文 PDF”，选择这些 PDF。
5. 插件解析 PDF 文本，并按题名、作者、文件名、PDF 首页文字自动匹配论文。
6. 用户导出 AI 分析包。
7. 如果只想快速看全局，上传快速摘要分析包；如果已经导入 PDF，则上传全文增强分析包，AI 会在同一批材料中优先依据全文、没有全文时依据摘要判断。

## 数据模型

在 `CnkiRecord` 上增加全文相关字段：

- `fullText`：提取出的全文纯文本。
- `fullTextFileName`：导入的 PDF 文件名。
- `fullTextImportedAt`：全文导入时间。
- `fullTextMatchScore`：自动匹配分数，范围 0 到 1。
- `fullTextStatus`：`none`、`matched`、`unmatched`、`failed`。

第一版不长期保存原始 PDF 文件，只保存提取后的文本和匹配信息。

## PDF 解析

浏览器插件本身没有稳定的 PDF 文本解析能力，需要引入 `pdfjs-dist`。解析流程：

1. 用户通过文件选择器选择 PDF。
2. 前端读取 `ArrayBuffer`。
3. 使用 PDF.js 逐页提取文字。
4. 合并为纯文本。
5. 对文本进行清洗：去掉多余空白、连续页眉页脚暂不做复杂判断。

## 自动匹配策略

自动匹配使用保守规则：

1. 优先用 PDF 文本前 3000 字匹配已采集论文题名。
2. 题名完全出现时直接匹配。
3. 题名未完全出现时，用归一化题名相似度匹配。
4. 文件名中包含题名主要片段时提高分数。
5. 作者出现在 PDF 首页文本中时提高分数。
6. 分数达到阈值才自动写入对应论文，低于阈值则列为未匹配。

第一版阈值设为 `0.72`。未匹配 PDF 不进入论文记录，但会在导出包中生成 `unmatched_fulltext.md`，方便用户知道哪些文件没有用上。

## 全文切片与导出

导出包不再把全部内容塞进一个 Markdown 文件，而是分层输出：

```text
papers.csv
papers.jsonl
01_快速摘要分析包.md
02_全文增强分析包/
  batch_001.md
  batch_002.md
02_全文论文索引.md
03_最终汇总提示词.md
analysis_prompt.md
merge_prompt.md
unmatched_fulltext.md
```

快速摘要分析包覆盖全部已采集论文。全文增强分析包也覆盖全部已采集论文：有全文的论文包含题录、摘要、关键词和全文切片；没有全文的论文保留题录、摘要、关键词，并标明“全文：未导入”。

全文增强批次按文本长度控制，不按固定篇数控制。第一版规则：

- 单篇全文按约 7000 个中文字符切片。
- 相邻切片保留约 500 字重叠。
- 每个全文增强批次总字符数控制在约 60000 字以内。

这样用户可以把每个批次单独上传给 AI，不需要一次上传全部全文，也不需要先让 AI 分析摘要、再分析全文。

## AI 使用方式

导出包中给出三类提示词：

1. `01_快速摘要分析包.md`：用于没有导入全文时的快速全局分析。
2. `02_全文增强分析包/batch_XXX.md`：用于一次性分析题录、摘要、关键词和可用全文；有全文优先依据全文，没有全文则只做摘要级判断。
3. `03_最终汇总提示词.md`：用于合并多个全文增强批次的分析结果，形成文献综述可用的结论。

## 用户界面

弹窗保持简单：

- `采集当前页并补全摘要`
- `导入全文 PDF`
- `导出 AI 分析包`
- `清空当前项目`

第一版导入时直接使用文件选择器，不新增复杂工作台页面。导入完成后状态显示：

```text
已读取 X 个 PDF，匹配 Y 篇，未匹配 Z 个，失败 N 个
```

## 测试策略

1. 单元测试 PDF 文本清洗和切片逻辑。
2. 单元测试论文匹配逻辑。
3. 单元测试导出包结构，确认快速摘要分析包和全文增强分析包分开。
4. 弹窗测试确认点击“导入全文 PDF”后会触发文件选择和保存记录。
5. 构建测试确认插件可以正常打包。

## 风险与边界

1. 部分 PDF 可能是扫描版图片，PDF.js 不能提取文字。第一版标记为失败，不做 OCR。
2. 自动匹配可能出错，所以阈值要保守。宁可未匹配，也不要错配。
3. 全文文本较长，会占用浏览器本地存储。第一版适合几十到一两百篇重点 PDF，不建议一次导入上千篇全文。
4. 不同 AI 平台上传文件大小限制不同，所以导出必须分批。
