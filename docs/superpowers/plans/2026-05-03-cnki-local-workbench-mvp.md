# 知网本地工作台 MVP 旧计划

## 状态

这份计划是早期“本地 Web 工作台”方向的历史记录。后续和用户讨论后，当前优先方向已经调整为“浏览器插件导出 AI 分析包”，因此本计划暂不执行。

## 原始思路

早期方案设想做一个本地 Web 应用，包含项目创建、检索策略辅助、题录导入、摘要分析、研究地图和文献矩阵导出等功能。这个方向的价值在于后续可以承接更长上下文、更复杂的文献综述整理工作。

## 暂停原因

用户明确指出当前唯一痛点是：知网检索结果很多，需要更快地把上千篇论文归纳总结出来。知网本身已经提供较好的检索能力，因此第一阶段不应把重点放在“生成检索策略”或“本地工作台界面”上，而应先解决“从知网页面采集题录并导出给 AI 分析”的问题。

## 后续可能恢复的条件

当浏览器插件能够稳定采集题录、摘要和关键词后，可以再恢复本地工作台方向，用于：

- 管理多个研究主题。
- 合并多批 AI 分析结果。
- 维护代表论文清单。
- 生成可追溯的文献综述草稿。
- 对接本地向量检索或长上下文模型。
